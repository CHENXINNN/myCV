动画/特效 作品展示

《猫神》

这是一个 svg 动画。动画运用到了大量的 svg 滤镜与 morph svg 技术。猫神眼圈周围的火焰效果是通过叠加 feGaussianBlur 与 feColorMatrix 滤镜完成的。鼻孔的喷气特效混合了 feTurbulence 与 feDisplacementMap 滤镜。眨眼则是借助 morph SVG 完成的。动画每秒有超过 280 个 DOM 在移动，但是通过优化依旧能达到 60 FPS。兼容到 IE10，能够很好的实现响应式。

《火烈鸟》

这个动画作品使用了一个叫做 ZDog 的库。ZDog 是一个伪 3D 的 JavaScript 引擎，其原理是在 canvas 或者 svg 上作画，通过视觉欺骗从而达到“看起来是3D”但实际上是 2D 的效果。因为不是真正的 3D 引擎，所以使用门槛低，适合平时制作一些简单的 3D 动画。但是也正因为 ZDog 是一个伪 3D 引擎，所以它无法很好的解决 z-fighting 的问题，在实际的业务中还需要谨慎使用。兼容性其实就是 canvas 和 svg 的兼容性，可以兼容到 IE9，也可以很好的实现响应式。

《滑动文字特效》

将粉色与黑色的字重合在一起，再利用 transform: skew() 将它们拉伸到对称的角度，然后使用 overflow: hidden 将两种颜色的文字各遮掉一半，再将 transform: translate() 的值与鼠标位置相关联，从而实现交互效果。没有使用到很新的 CSS 属性，所以可以兼容到 IE9。可以很好的实现响应式。

《剪切文字特效》

这个特效其实使用了三层相同的文字。当鼠标悬浮在文字上时，顶层的文字被一个与背景颜色相同的 div 元素插入，然后将 opacity 减到 0 实现渐变消失。 第二层和第三层的文字利用 clip-path 让第二层的文字只显示左半边，让第三层的文字只显示右半边，在鼠标悬停的时候让右半边文字位移一段距离，从而实现剪切效果。由于使用了 clip-path， 不兼容 IE 与使用旧内核的 Edge 浏览器，可以很好的实现响应式。

《阴影》

这是一个使用了 Three.js 的 WebGL 动画，几何模型本身很简单，但是通过色彩的运用、光与阴影的结合，营造出了一种极简的美感。用户可以使用鼠标转动模型与模型交互。兼容到 IE11，能够很好的实现响应式。

《魔方》

这个动画是借助 css 3d transform 和 svg 实现的。由于 css 的 3D transform不能直接作用于 svg 元素 (因为 svg 有自己独立的坐标系，而且是二维的)，所以需要在 svg 外面包裹一层 div，然后把 3d transform 作用在 div 上，从而间接的实现 svg 的三维变换。由于使用了 css 3d transform， 所以不兼容 IE。能够很好的实现响应式。

《3D卡片特效》

一个利用 css 3d transform 完成的卡片展示特效，可以根据鼠标的移动展示卡片的不同角度。所以不兼容 IE。能够很好的实现响应式。

《知乎 logo 动画》

动画的前半段是一个经典的 svg 路径动画，通过调节 svg 的 stroke-dashoffset 与 stroke-dashoffset 实现。后半段是通过 svg 滤镜实现的一个霓虹效果。兼容到 IE10，能够很好的实现响应式。

《月相》

这是一个使用了自定义 glsl 语言的 WebGL 动画。球体表面的黑白渐变是通过 glsl 着色器语言编写 vertex shader 和 fragment shader 实现的，也就是说这个黑白渐变这就是球体本身的材料颜色，而不是通过光源与阴影实现的。兼容到 IE11，能够很好的实现响应式。

《通过 border radius 实现的流体特效》

border-radius 最多可以接收八个参数，配合 css animation, 不需要使用 svg, 就可以实现很多有趣的流体特效。兼容到 IE10，可以很好的实现响应式。

《阴阳加载动画》

动画本身并不复杂，难的地方在于怎么样把这个动画做成响应式的同时保证八卦图的长宽比不变。最后发现借助 svg 的 preserveAspectRatio="xMidYMid slice" 可以实现这个效果: 左右两扇门的实际大小比 viewport 要大，随着窗口的缩小只居中显示一部分svg。兼容到 IE9，可以很好的实现响应式。

《视差滚动特效》

视差滚动特效第一次出现在2014年，如今我们在很多网站上都可以看到它的身影。它的原理是通过前景与背景移动速度的差别，营造出一种奇妙的视觉体验。视差滚动一般分为两种：一种是前景和背景都动，但是前景比背景动得快；还有一种是只有前景动，背景不动。前者既可以通过 js 实现，也可以通过 css 3d transform 实现。js实现的兼容性比较好，css 实现的性能比较好。后者主要通过 background-attachment: fixed 实现，比较简单。GIF 展示的是第一种。
